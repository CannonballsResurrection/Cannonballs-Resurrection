How Can I Tell If My Firewall/Proxy Is Causing Problems?

  • When you create a game, no players will be able to join (or only some).
  • When attempting to join a game, I see a 'Connection Failure' message.

Firewalls

This game is reliant on your Internet connection being fully functional. Firewalls and proxy servers are designed to block your Internet connection in order to minimize the chances that your computer will be accessed by others. They also prevent you from playing games online, since most games need a direct connection between your computer and others to function. If you are using a firewall or proxy, Cannonballs!TM may not function properly.

Connecting to different computers requires access through ports, and firewalls and proxies will close most ports to protect your computer. If these ports are closed, you will need to find a way to open them, and this document contains many tips for doing so.

Port Forwarding

You may have to configure your firewall, proxy, or NAT to "forward" packets sent to the ports listed below to a specific IP address within the home network. If this is the case, the computer on which you wish to run the game should be the destination for the port forwarding.

Firewall Software

Here are some common firewalls/proxies-

  • Black Ice
  • Microsoft Proxy Server 2.0
  • Zonealarm
  • Norton Internet Security
  • Tiny Personal Firewall
  • Sygate
  • Wingate
  • Internet Connection Sharing with Windows 98SE or higher
  • Linksys Routers
  • Cisco Routers

Note: Opening ports for TCP/IP gaming is a security risk, especially if you completely disable your firewall system. If you are using a dial-up connection, the security risk is low, but if you have a cable modem or DSL connection, you should be careful when allowing exceptions to the default port blocks. For more information, please consult the documentation of your networking software.

Also consult the documentation provided with your networking software to determine the steps to open these ports. Some examples are given below.

After having played, please remember to turn on the firewall or close the ports you temporarily enabled for playing.

Working Around The Firewall / Proxy

If you are behind a firewall/proxy and can change your firewall settings, Cannonballs!TM needs the following ports open:

  • 47624 TCP Outbound
  • 47624 TCP Inbound
  • 2300-2400 TCP Inbound
  • 2300-2400 TCP Outbound
  • 2300-2400 UDP Inbound
  • 2300-2400 UDP Outbound

Help! Disabling Firewalls Doesn't Work!

Usually you cannot simply disable your firewall or proxy and expect the game to play. Turning off your firewall/proxy still leaves the ports closed. Additionally, other factors may interfere with your connection. To work around these issues, you will need to consult the manufacturer of your firewall/proxy.

Disabling / Permission On Specific Firewall Programs

We do not recommend disabling your firewall. Disabling it will not make it open up the ports it has already closed. Instead, you need to configure it to allow access to Cannonballs!TM. Proxies/Firewalls generally call this "giving permission," and some possibile ways to do it are listed below.

WindowsXP Internal Firewall

WindowsXP has a built-in firewall. If you are using another firewall or proxy with WindowsXP, you will have to speak with the firewall/proxy manufacturer and Microsoft to determine their compatibility.

For Windows XP's built-in firewall

  • Open Network Connections (Click Start, click Control Panel, and then double click Network Connections.)
  • Click the Dial-up, LAN or High-Speed Internet connection that you want to open for multiplayer games, and then, under Network Tasks, click Change Settings of this connection.
  • On the Advanced tab, under Internet Connection Firewall, clear the Protect my computer and network by limiting or preventing access to this computer from the Internet check box.

Universities

Most Universities are protected by firewalls which cannot be worked around on a user-by-user basis. You should consult an IT representative at your University to see if any workaround is possible.

Microsoft Proxy Server 2.0

  • Internet Service Manager, Windows Sockets Proxy service: Service Properties
  • Add protocol "DirectPlay (client)" under Protocols tab.
  • Add protocol "DirectPlay (host)" under Protocols tab.
  • Initial connection
  • 47624 TCP Outbound
  • 47624 TCP Inbound
  • Subsequent connections
  • 2300-2400 TCP Inbound
  • 2300-2400 TCP Outbound
  • 2300-2400 UDP Inbound
  • 2300-2400 UDP Outbound
  • Give appropriate permissions to "DirectPlay (client)" under Permissions tab.
  • Give appropriate permissions to "DirectPlay (host)" under Permissions tab.
  • Add to file "mspclnt.ini", which is in the C:\Msp\Clients folder by default.
    [Common Configuration]
    RemoteBindUdpPorts=2300-2400
    ServerBindTcpPorts=2300-2400
    KillOldSession=1

Black Ice

Set the security level to low.

ZoneAlarm

It is best to make ZoneAlarm recognize MSIE as an allowable application. When you attempt to join or create a game of Cannonballs!TM and ZoneAlarm alerts you that MSIE is trying to access the Internet, you should allow it to do so, and check the box to "remember this application." You should still leave ZoneAlarm on medium to high security.

You can also set ZoneAlarm to recognize MSIE as a part of your Local Network and set both local and Internet settings to "Low."

Norton Internet Security 2002

NIS2002 is very simple - you only need to give MSIE permission. Don't try to provide Custom Settings. When you are notified that MSIE is attempting to access the Internet, check 'Permit this application to access the Internet' and press OK.

F-Secure Distributed Firewall

First activate the Custom profile so that you can add customized configuration rules, and then add services that allow "TCP High ports" and "UDP high ports" for both inbound and outbound traffic. Note that this configuration opens all ports above 1023 to both inbound and outbound traffic. If allowed by your software, you should oonly open the ports listed earlier in this document.

Internet Connection Sharing

  • Go to the Network Properties on the modem that is being shared
  • Click on the Sharing tab (note that you should see Internet Connection Sharing is checked, if you have the right one)
  • Click on the Settings button
  • Click on Add and add as follows:
    • Name: directplay
    • Remote server port number: 2300
    • Radio button for TCP selected
    • TCP port range: 2300-2400
    • UDI port range: 2300-2400
  • Click on OK to add it
  • Repeat, adding another one, as above, but with the UDP radio button instead
  • Click OK - it should say you will need to reconnect for the changes to work

Linksys Router

  • Go to its config page - use a browser and go to http://192.168.1.1 - the default user is blank, default password is admin
  • From there, go to the Advanced page
  • From there, go to the Port Forwarding page
  • From there, go to the Port Triggering page
  • In there, add entries as follows:
    • name: directplay
    • port type: both (choices of udp, tcp or both)
    • port range: 2300-2400
  • Click on apply

Other Firewalls with Per-Program based control

If you use a personal firewall that is based on a per-program based control of the use of ports, just enable the needed ports as you are notified by the firewall program of MSIE's attempt to access these ports. Since Cannonballs!TM plays in a browser window, instead of as a standalone application, you need to ensure that the browser has the ability to access the ports.
